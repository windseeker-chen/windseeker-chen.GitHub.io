438.找到字符串中所有字母异位词

12 Nov 2024

题目说明

给定两个字符串 s 和 p，找到 s 中所有 p 的 异位词的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

  • 示例 1:
    输入: s = “cbaebabacd”, p = “abc”
    输出: [0,6]
    解释:
    起始索引等于 0 的子串是 “cba”, 它是 “abc” 的异位词。
    起始索引等于 6 的子串是 “bac”, 它是 “abc” 的异位词。
  • 示例 2:
    输入: s = “abab”, p = “ab”
    输出: [0,1,2]
    解释:
    起始索引等于 0 的子串是 “ab”, 它是 “ab” 的异位词。
    起始索引等于 1 的子串是 “ba”, 它是 “ab” 的异位词。
    起始索引等于 2 的子串是 “ab”, 它是 “ab” 的异位词。

提示:

  • 1 <= s.length, p.length <= 3 * 104
  • s 和 p 仅包含小写字母

方法一：暴力遍历

  1. 主循环遍历[：len(s)-len(p)],获取到和p长度一致的substring
  2. 判断substring和p是否是异位词
    1. 分别构建string的字符计数hash
    2. 判断hash是否相等

Code

func findAnagrams(s string, p string) []int {
	result := make([]int, 0)
	hashP := getHash(p)
	for i := 0; i <= len(s)-len(p); i++ {
		sub := s[i : i+len(p)]
		hashSub := getHash(sub)
		if isAnagram(hashSub, hashP) {
			result = append(result, i)
		}
	}

	return result
}

func isAnagram(s, t map[byte]int) bool {
	if len(s) != len(t) {
		return false
	}
	for char, count := range s {
		if count != t[char] {
			return false
		}
	}
	return true
}

func getHash(s string) map[byte]int {
	result := make(map[byte]int, len(s))
	for i := 0; i < len(s); i++ {
		result[s[i]]++
	}
	return result
}

复杂度分析

定义s的长度为N，p的长度为M

时间复杂度

  1. 主循环需要执行O(N-M)
  2. 每次异位词判断
    1. 计算substring hash需要O(M)
    2. 比较hash需要O(M) 总时间复杂度为O((N-M)*2M)

空间复杂度

  1. p的hash需要O(M)
  2. 每次异位词判断都要
    1. substring存储hash需要O(M) 总空间复杂度为O(M+(N-M)*M)

方法二：优化hash存储

主循环遍历时，我们可以发现，假设上一次子串起点为i，新字串只需要移除s[i]，添加s[i+len(p)+1]，所以我们可以只维护一个字串的hash

Code

func findAnagrams(s string, p string) []int {
	if len(p) > len(s) {
		return []int{}
	}
	result := make([]int, 0)
	hashP := getHash(p)
	hashS := getHash(s[0:len(p)])
	for i := 0; i <= len(s)-len(p); i++ {
		if isAnagram(hashP, hashS) {
			result = append(result, i)
		}
		if i+len(p) < len(s) {
			hashS[s[i+len(p)]]++
			hashS[s[i]]--
			if hashS[s[i]] == 0 {
				delete(hashS, s[i])
			}
		}
	}

	return result
}

func getHash(s string) map[byte]int {
	result := make(map[byte]int, len(s))
	for i := 0; i < len(s); i++ {
		result[s[i]]++
	}
	return result
}

func isAnagram(s, t map[byte]int) bool {
	if len(s) != len(t) {
		return false
	}
	for char, count := range s {
		if count != t[char] {
			return false
		}
	}
	return true
}

复杂度分析

定义s的长度为N，p的长度为M

时间复杂度

  1. 主循环需要执行O(N-M)
  2. 每次异位词判断
    1. 计算substring hash需要O(1)
    2. 比较hash需要O(M) 总时间复杂度为O((N-M)*M)

空间复杂度

  1. p的hash需要O(M)
  2. substring存储hash需要O(M) 总空间复杂度为O(N)

方法三：优化异位词判断

上面一直使用hash是否相等来判断异位词，可以注意到题目描述中只包含小写字母，我们可以使用数组[26]int来替代hash，达到节省存储和提高比较速度(数组特性和编译器优化)

Code

func findAnagrams(s string, p string) []int {
	if len(p) > len(s) {
		return []int{}
	}
	result := make([]int, 0)
	pCount, sCount := [26]int{}, [26]int{}
	for i := 0; i < len(p); i++ {
		pCount[p[i]-'a']++
		sCount[s[i]-'a']++
	}
	for i := 0; i <= len(s)-len(p); i++ {
		if pCount == sCount {
			result = append(result, i)
		}
		if i+len(p) < len(s) {
			sCount[s[i+len(p)]-'a']++
			sCount[s[i]-'a']--
		}
	}
	return result
}

复杂度分析

定义s的长度为N，p的长度为M

时间复杂度

  1. 计算字符数组需要执行O(M)
  2. 主循环需要执行O(N-M)
  3. 每次异位词判断时间固定值，定为O(1) 总时间复杂度为O(N)

空间复杂度

  1. 2个固定大小的字符数组，定为O(1) 总空间复杂度为O(1)

方法四：双指针优化

方法三中采用定长的滑动窗口截取子串来比较，可以采用双指针的不定长滑动窗口来避免不必要的全量对比，提前剪枝.

  1. 定义窗口的左/右端点
  2. 外层遍历右端点，如果右端点的字符频率和p不一致，移动左端点直至一致
  3. 当发现字串和p长度一致且字符频率一致，记入结果

为什么只需要判断右端点？
外层循环只改动了右端点，当右端点移动时，窗口内的其他字符频率没有变化，所以只需要关注新加入的右端点字符的频率

Code

func findAnagrams(s string, p string) []int {
	if len(p) > len(s) {
		return []int{}
	}
	result := make([]int, 0)
	pCount := [26]int{}
	for i := 0; i < len(p); i++ {
		pCount[p[i]-'a']++
	}
	left := 0
	for right, char := range s {
		c := char - 'a'
		pCount[c]--
		for pCount[c] < 0 {
			pCount[s[left]-'a']++
			left++
		}
		if right-left+1 == len(p) {
			result = append(result, left)
		}
	}
	return result
}

注：不定长滑动窗口思路

  1. 维护一个有条件的窗口
  2. 移动右端点，扩大窗口，导致不满足条件
  3. 移动左窗口，缩小窗口，重新满足条件 本题中，窗口条件是窗口形成的字串的长度和字符频率与p一致
